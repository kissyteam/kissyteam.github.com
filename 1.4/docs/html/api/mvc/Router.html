Parameters:config (Object) – 配置项.详细见下方 Configs Detail .

Configs Detail¶

Router.config.routes¶

{Object} - 可选，路由策略和规则

{
    "":"index",
    "/edit/:id":function(paths,query){
        // /editor/99?q=1
        alert(paths.id);  // => 99
        alert(query.q); // => 1
    },
    "/new/*u":function(paths){
       // /new/kissy-mvc-is-good/right
       alert(paths.u); // => kissy-mvc-is-good/right
    },
    "test_regexp":{
        reg:/\/list-(\w)/,
        callback:function(matches,query){
            // /list-t
            alert(matches[0]); // => t
        }
    }
}

用于指定当 url 符合一定规则时执行指定的函数，当函数为字符串时表示路由实例上的函数，规则有 3 种

  1. :id 用于匹配 url / 间部分
  2. *u 用户匹配从当前位置到结尾的 url 部分
  3. 直接配置正则表达式

调用函数形式为

Router.config.routes.fn()¶
fn( paths [ , querys ] )
Parameters:
  • paths (Object) – 包含规则路径参数和当前 url 匹配值为键值对的对象。当规则直接配置为正则表达式时，该参数值为匹配正则式的分组数组。
  • querys (Object) – 包含当前url的查询参数和值为键值对的对象

Router.navigate()¶

    Router.navigate.opts.triggerRoute¶

    {boolean} - 默认 false。是否强制触发对应该路径的路由处理。默认如果当前路径和 path 相同则不会发生任何动作.
Router.start()¶
start( [ opts ] )
开始路由规则处理，一个页面仅限入口调用一次
Parameters:opts (Object) –

包括以下控制选项

Router.start.opts.triggerRoute¶

{boolean} - 默认 false。是否触发当前路径的路由处理函数，默认只有当下次 url 发生变化时才处理.

Router.Events.route:x()¶
route( e )

  • e.paths (Object) – 同 fn() 中 paths 格式
  • e.query (Object) – 同 fn() 中 query 格式
